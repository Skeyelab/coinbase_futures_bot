## Accessing the Bot

The bot exposes three primary interfaces. Choose the one that fits your workflow:

| Interface | Best For | How to Start |
|-----------|----------|--------------|
| **`bin/futuresbot` CLI** | TUI dashboard, chat, quick status/position/signal views | `bin/futuresbot` |
| **AI Chat Bot** | Interactive natural-language control | `bin/futuresbot chat` |
| **REST API** | Programmatic or scripted access | `curl http://localhost:3000/…` |
| **Rake Tasks** | One-off operations from the terminal | `bin/rake <task>` |
| **Web UI** | Viewing and editing positions visually | `http://localhost:3000/positions` |
| **Slack** | Notifications and quick commands | See [Slack Integration](#slack-integration) |

### Prerequisites

Before using any interface, make sure the bot is running:

```bash
# 1. Start the Rails server (required for API and Web UI)
bin/rails server

# 2. Start the background job worker (required for signals and automated tasks)
bundle exec good_job start

# 3. Optionally start the real-time signal system
bin/rake realtime:signals
```

Health check — the server is ready when this returns `{"status":"ok"}`:

```bash
curl http://localhost:3000/up
```

## Analyzing the Market

### Using the AI Chat Bot

The easiest way to get a market snapshot is through the conversational chat bot:

```bash
bin/futuresbot chat
```

Example queries:

```
FuturesBot> BTC price
FuturesBot> show ETH market data
FuturesBot> what signals are active?
FuturesBot> show recent trading alerts
FuturesBot> any new entry signals?
FuturesBot> what should I do with BTC?
FuturesBot> analyze BTC market conditions
```

The bot will respond with price data, active signals, technical indicator readings, sentiment scores, and a trading recommendation including entry/exit points.

### Viewing Active Signals via API

Signals represent actionable trade recommendations generated by the multi-timeframe strategy engine. Use the REST API to query them:

```bash
# All currently active signals (requires API key)
curl -H "X-API-Key: $SIGNALS_API_KEY" http://localhost:3000/signals/active

# Only high-confidence signals (≥80% confidence)
curl -H "X-API-Key: $SIGNALS_API_KEY" \
  "http://localhost:3000/signals/high_confidence?threshold=80"

# Signals from the last hour
curl -H "X-API-Key: $SIGNALS_API_KEY" \
  "http://localhost:3000/signals/recent?hours=1"

# Filter by symbol and direction
curl -H "X-API-Key: $SIGNALS_API_KEY" \
  "http://localhost:3000/signals?symbol=BTC-USD&side=long&min_confidence=70"

# Signal statistics for the past 24 hours
curl -H "X-API-Key: $SIGNALS_API_KEY" \
  "http://localhost:3000/signals/stats?period=24h"
```

A signal response looks like:

```json
{
  "id": 123,
  "symbol": "BTC-USD",
  "side": "long",
  "confidence": 85.5,
  "entry_price": 45000.00,
  "stop_loss": 44500.00,
  "take_profit": 45800.00,
  "quantity": 2,
  "timeframe": "5m",
  "alert_status": "active",
  "alert_timestamp": "2025-01-18T10:25:00Z",
  "expires_at": "2025-01-18T11:25:00Z",
  "metadata": {
    "ema_trend": "bullish",
    "sentiment_z_score": 1.2
  }
}
```

> **Tip:** Set `SIGNALS_API_KEY` in your `.env` file. Pass it via the `X-API-Key` header or `?api_key=` query parameter.

### Viewing Sentiment Data

Sentiment scores are derived from CryptoPanic news headlines and augment the technical signals:

```bash
# Recent BTC sentiment (15-minute windows)
curl "http://localhost:3000/sentiment/aggregates?symbol=BTC-USD&window=15m"

# ETH sentiment over the past 10 hours (1-hour windows)
curl "http://localhost:3000/sentiment/aggregates?symbol=ETH-USD&window=1h&limit=10"
```

Sentiment fields:
- `avg_score` — raw average sentiment (−1 bearish → +1 bullish)
- `weighted_score` — recency-weighted score
- `z_score` — how far the current score is from its historical mean (used for signal gating)

### Triggering Signal Evaluation Manually

To generate a fresh set of signals on demand:

```bash
# Trigger evaluation for all enabled trading pairs
curl -X POST -H "X-API-Key: $SIGNALS_API_KEY" \
  -H "Content-Type: application/json" \
  http://localhost:3000/signals/evaluate

# Trigger for specific symbols
curl -X POST -H "X-API-Key: $SIGNALS_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"symbols": ["BTC-USD", "ETH-USD"]}' \
  http://localhost:3000/signals/evaluate
```

Or use rake tasks:

```bash
# Evaluate all pairs
bin/rake realtime:evaluate

# Evaluate a single symbol
bin/rake realtime:evaluate_symbol[BTC-USD]

# View signal statistics
bin/rake realtime:stats
```

### Subscribing to Real-Time Signal Updates (WebSocket)

For live alerts, connect to the Action Cable channel:

```javascript
const ws = new WebSocket('ws://localhost:3000/cable');

ws.onopen = () => {
  ws.send(JSON.stringify({
    command: 'subscribe',
    identifier: JSON.stringify({ channel: 'SignalsChannel' })
  }));
};

ws.onmessage = (event) => {
  const data = JSON.parse(event.data);
  if (data.type === 'signal_alert') {
    console.log('New signal:', data.signal);
  }
};
```

### Collecting Market Data

Pull fresh historical candles to ensure signals are computed from up-to-date data:

```bash
# Fetch the latest hourly candles for all pairs
bin/rake market_data:backfill_1h_candles[1]

# Subscribe to live market data for specific symbols
bin/rake market_data:subscribe[BTC-USD]

# Subscribe to multiple symbols
PRODUCT_IDS=BTC-USD,ETH-USD bin/rake market_data:subscribe
```

## Placing Trades

> ⚠️ **Important**: Always test in paper trading mode (`PAPER_TRADING_MODE=true`) before placing live trades.

### Using the AI Chat Bot

The chat bot supports natural-language trading control commands:

```bash
bin/futuresbot chat
```

**Start / Resume Trading**

```
FuturesBot> start trading
✅ Trading has been activated. The bot will now generate signals and manage positions.

FuturesBot> resume trading
```

**Stop / Pause Trading**

```
FuturesBot> stop trading
⏸️ Trading has been paused.

FuturesBot> pause trading operations
```

**Check Position Sizing Before Trading**

```
FuturesBot> position sizing
📊 Position Sizing Configuration:
Equity: $10,000.00
Risk per trade: 2.0%
Max risk per trade: $200.00
```

### Configuring Your Trading Capital

Set these environment variables before starting:

```bash
# Your account equity (used for position sizing)
export SIGNAL_EQUITY_USD=5000

# Risk per trade as a percentage of equity (default: 2%)
export RISK_PER_TRADE_PERCENT=2

# Minimum signal confidence to act on (0–100, default: 60)
export REALTIME_SIGNAL_MIN_CONFIDENCE=65
```

### Starting the Real-Time Trading System

Once configured, start the full real-time system:

```bash
SIGNAL_EQUITY_USD=5000 bin/rake realtime:signals
```

This continuously:
1. Subscribes to live Coinbase price feeds
2. Aggregates real-time candles (1m/5m/15m/1h)
3. Evaluates the multi-timeframe strategy every 30–45 seconds
4. Broadcasts signals and (if `PAPER_TRADING_MODE=false`) executes trades

### Generating Signals in Batch Mode

For scheduled or one-off signal generation:

```bash
# Generate signals for all enabled pairs
bin/rake signals:run

# Or using a runner
bin/rails runner "GenerateSignalsJob.perform_now(equity_usd: 10000)"
```

### Paper Trading (Simulation)

Test your configuration without risking real funds:

```bash
# Ensure paper trading mode is on
export PAPER_TRADING_MODE=true

# Run a single paper trading step
bin/rake paper:step

# Check simulated positions and PnL
bin/rake day_trading:pnl
```

### Executing a Manual Signal

To manually mark a signal as triggered (e.g., you executed it yourself):

```bash
curl -X POST -H "X-API-Key: $SIGNALS_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"execution_price": 45050.00, "notes": "Manual execution"}' \
  http://localhost:3000/signals/123/trigger
```

## Watching Positions

### Using the AI Chat Bot

```bash
bin/futuresbot chat
```

Position queries:

```
FuturesBot> show my positions
📊 Positions Summary
Open: 2 (Day: 1, Swing: 1)
Total PnL: $156.78
BTC-PERP: +$98.45 (Day Trading)
ETH-PERP: +$58.33 (Swing Trading)

FuturesBot> what's my current P&L?
FuturesBot> how many positions are open?
FuturesBot> display trading summary
```

### Using the Web Interface

Navigate to the position dashboard in your browser:

```
http://localhost:3000/positions
```

From the web UI you can:
- View all open positions with entry price and current P&L
- Create, edit, or close positions manually
- Increase position size

### Using the REST API

```bash
# All open positions
curl http://localhost:3000/api/positions

# Day trading positions only
curl "http://localhost:3000/api/positions?type=day_trading"

# Positions for a specific symbol
curl "http://localhost:3000/api/positions?product_id=BTC-USD"

# Aggregated position summary (total PnL, by symbol)
curl http://localhost:3000/api/positions/summary

# Position exposure and risk metrics
curl http://localhost:3000/api/positions/exposure
```

Example position summary response:

```json
{
  "total_positions": 4,
  "day_trading_positions": 3,
  "swing_positions": 1,
  "total_unrealized_pnl": 1250.00,
  "total_realized_pnl_today": 850.00,
  "by_symbol": {
    "BTC-USD": { "positions": 2, "unrealized_pnl": 950.00 },
    "ETH-USD": { "positions": 2, "unrealized_pnl": 300.00 }
  }
}
```

### Using Rake Tasks

```bash
# Check all day trading position statuses
bin/rake day_trading:check_positions

# View current profit/loss
bin/rake day_trading:pnl

# View detailed position breakdown
bin/rake day_trading:details

bin/rake day_trading:manage
```

tail -f log/development.log | grep -E "Position|PnL|DayTrading"

## Managing Risk

### Automatic Risk Controls

The bot enforces these controls automatically:

| Control | Behaviour |
|---------|-----------|
| **Take Profit** | Position closes when price hits the take-profit level |
| **Stop Loss** | Position closes when price hits the stop-loss level |
| **24-Hour Limit** | Day-trading positions are force-closed after 24 hours |
| **End-of-Day Closure** | All day-trading positions closed at midnight UTC |
| **Rate Limiting** | Max signals per hour (`REALTIME_SIGNAL_MAX_PER_HOUR`) |

### Checking Take Profit / Stop Loss Triggers

```bash
# Check all positions for TP/SL hits
bin/rake day_trading:check_tp_sl

# Close any positions that have hit their TP/SL
bin/rake day_trading:close_expired
```

### Adjusting Risk Parameters

Edit your `.env` file or export environment variables:

```bash
# Conservative settings
export SIGNAL_EQUITY_USD=1000
export RISK_PER_TRADE_PERCENT=1
export REALTIME_SIGNAL_MIN_CONFIDENCE=75
export REALTIME_SIGNAL_MAX_PER_HOUR=5

# Moderate settings (10-contract comfort zone)
export SIGNAL_EQUITY_USD=5000
export RISK_PER_TRADE_PERCENT=2
export REALTIME_SIGNAL_MIN_CONFIDENCE=65
export REALTIME_SIGNAL_MAX_PER_HOUR=8
```

## Slack Integration

If Slack is configured (see `SLACK_SETUP.md`), you can use slash commands directly in your workspace:

| Command | Description |
|---------|-------------|
| `/bot-status` | Current trading status, open positions, daily PnL |
| `/bot-positions` | All open positions |
| `/bot-positions BTC` | Positions filtered by symbol |
| `/bot-pnl today` | Today's profit/loss report |
| `/bot-pnl week` | Weekly PnL report |
| `/bot-health` | System health status |
| `/bot-pause` | Pause new signal generation |
| `/bot-resume` | Resume trading operations |
| `/bot-stop` | 🚨 Emergency stop — closes all positions |
| `/bot-help` | Full command list |

You can also mention the bot in any channel:

```
@FuturesBot what's the status?
@FuturesBot help
```

## Emergency Controls

### Stop All Trading Immediately

**Via Chat Bot:**

```
FuturesBot> emergency stop
🚨 EMERGENCY STOP EXECUTED 🚨
All trading activities have been immediately stopped.
Positions closed: 2 | Orders cancelled: 0
```

**Via Rake Task:**

```bash
FORCE=true bin/rake realtime:cancel_all
```

**Via Slack:**

```
/bot-stop
```

### Force-Close All Day Trading Positions

```bash
bin/rake day_trading:force_close_all
```

### Cancel All Active Signals

```bash
# Cancel all active signals via rake task
FORCE=true bin/rake realtime:cancel_all
```
